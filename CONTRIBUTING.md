# Contributing

Contributions are welcome! This skill improves through real-world DevOps experience.

## What to Contribute

- Corrections to existing guidance
- New patterns for Terraform, Kubernetes, Azure, or AWS
- Additional CI/CD pipeline templates
- Troubleshooting entries for common issues
- New scripts or template files

## How to Contribute

1. Fork the repository
2. Create a branch: `git checkout -b feat/your-topic`
3. Make your changes
4. Ensure SKILL.md stays focused — move detailed content to `references/`
5. Open a Pull Request with a clear description of what you added and why

## Guidelines

- Keep reference files focused on a single topic
- Prefer actionable patterns over theory
- Include concrete examples (HCL, YAML, shell) where possible
- Avoid duplicating official documentation — link to it instead
- Test scripts before submitting

## Reporting Issues

Open a GitHub Issue if you find outdated guidance, incorrect examples, or missing topics.
